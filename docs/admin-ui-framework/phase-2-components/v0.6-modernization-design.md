## 2. 视觉系统现代化

### 2.1 Token 重构目标

**当前问题：**
- 阴影过浅，层级感弱
- 圆角偏小，缺乏现代感
- 字体层级不够明确

**改进方案：**

```css
/* 阴影升级 - 更柔和的多层阴影 */
--ra-shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.03), 0 1px 3px 0 rgb(0 0 0 / 0.05);
--ra-shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.05), 0 2px 4px -2px rgb(0 0 0 / 0.03);
--ra-shadow-lg: 0 10px 15px -3px rgb(0 0 0 / 0.06), 0 4px 6px -4px rgb(0 0 0 / 0.04);
--ra-shadow-xl: 0 20px 25px -5px rgb(0 0 0 / 0.07), 0 8px 10px -6px rgb(0 0 0 / 0.04);

/* 圆角升级 */
--ra-radius-sm: 6px;   /* 4 -> 6 */
--ra-radius-md: 8px;   /* 6 -> 8 */
--ra-radius-lg: 12px;  /* 8 -> 12 */
--ra-radius-xl: 16px;  /* 12 -> 16 */

/* 新增 surface 层级 */
--ra-color-bg-surface-secondary: var(--ra-primitive-neutral-2);
--ra-color-bg-surface-tertiary: var(--ra-primitive-neutral-3);

/* 新增 elevated 层级 */
--ra-color-bg-elevated-soft: rgba(255, 255, 255, 0.8);
```

## 3. 图标体系现代化

### 3.1 选型决策

**选用 lucide-vue-next**

理由：
- 线性风格统一，现代感强
- 24x24 基准，清晰度高
- 线宽 2px 标准化
- 2000+ 图标覆盖所有场景
- 开源、轻量、Tree-shakeable

### 3.2 封装方案

```typescript
// src/components/icon/RIcon.vue
export interface RIconProps {
  name: string           // 图标名称
  size?: number | string // 尺寸 (16/18/20/24/32)
  color?: string         // 颜色 (语义 token 或 CSS 变量)
  strokeWidth?: number   // 线宽 (1.5/2/2.5)
}
```

### 3.3 迁移策略

1. 安装 lucide-vue-next 依赖
2. 创建 RIcon 封装组件
3. 创建图标映射表（旧名 → 新名）
4. 批量替换 UI 框架内图标
5. 批量替换 Web 应用内图标
6. 提供 icon usage guideline 文档
