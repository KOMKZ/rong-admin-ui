# 迭代日志 v0.2

- 迭代范围：Phase 1 Hardening + Phase 2 组件能力标准化
- 日期：2026-03-05

## 完成清单

### Phase 1 Hardening（阻断级缺陷修复）

1. **http-client 错误语义透传**
   - 新增 `RequestErrorKind` 类型枚举（HTTP_ERROR / NETWORK_ERROR / TIMEOUT / CANCELLED）
   - `RequestError` 增加 `kind` 字段，所有错误路径均赋予明确 kind
   - timeout 使用独立 AbortController，通过 `mergeSignals` 与外部 signal 组合
   - `classifyAbortError` 区分 timeout vs 外部取消

2. **token-manager 幂等刷新**
   - `startAutoRefresh` 开头调用 `stopAutoRefresh()`，防止叠加 timer
   - `init()` 增加 `initialized` 标记，重复调用幂等
   - 跨 Tab 同步完善：login/token_refreshed 事件现在会启动 auto-refresh
   - `destroy()` 重置 `initialized`，允许重新 init

3. **bootstrap 生命周期清理**
   - 收集所有 window.addEventListener 的清理函数
   - destroy 时依次执行清理 + 清空 errorHandler
   - 增加 `destroyed` 标记防止重复销毁

4. **路由 contract 对齐**
   - `notFoundComponent` 在 route-generator 中落地：叶子节点无 component 或无匹配 viewModule 均回退
   - route 构建逻辑重构为更清晰的条件分支
