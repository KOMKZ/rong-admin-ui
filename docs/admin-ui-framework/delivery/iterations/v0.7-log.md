### B. 商业化组件增强

#### B1. RStatCard 环比/同比对比

- 新增 `StatCardCompare` 接口：`previousValue`、`currentValue`、`period`、`periodLabel`
- 自动计算百分比变化和趋势方向
- 支持 `day/week/month/quarter/year/custom` 6 种周期
- 显示上期值对比区域
- 向后兼容：`trend`/`trendValue` 显式传值优先于 `compare` 自动计算
- 新增 22 个单测（含 7 个 compare 专项）

#### B2. RDrawerForm 异步校验 + 错误恢复

- 新增 `asyncValidator` prop：异步校验函数，返回 `DrawerFormError[]`
- 新增错误摘要面板（`role=alert`，含字段名+错误信息）
- 新增 Retry 按钮（提交失败后可见，emit `retry` 事件）
- 异常捕获：asyncValidator 抛出异常时自动降级为错误摘要
- 编辑表单时自动清除错误
- 关闭抽屉时自动重置错误状态
- 新增 `clearErrors` expose 方法
- 新增 19 个单测（含 asyncValidator、error recovery、expose）

## 失败模式 & CAPA

### FP-1: CSS dist 直链导致样式丢失

| 项 | 内容 |
|----|------|
| 现象 | 消费方页面裸样式，CSS 变量全部未生效 |
| 根因 | `main.ts` 使用 `@rong/admin-ui/dist/admin-ui.css` 而非 `@rong/admin-ui/style.css` |
| 纠正 | 修改为 `style.css` + gate-consumption 检测 |
| 预防 | 硬门禁 `css-import-check` + E2E `style-load-verification` |
| 验证 | gate-consumption PASS + style-load E2E PASS |

### FP-2: Naive UI 组件名与 stub 名不匹配

| 项 | 内容 |
|----|------|
| 现象 | 单测中 Naive UI 组件 stub 无效，Teleport 导致空渲染 |
| 根因 | Naive UI 导出的 `NDrawer` 实际 name 为 `Drawer`（无 N 前缀） |
| 纠正 | stub 键使用 `Drawer`/`DrawerContent`/`Button`/`Spin`/`Space` |
| 预防 | 组件测试模板中明确 Naive UI 组件名映射 |
| 验证 | drawer-form 19 tests PASS |
