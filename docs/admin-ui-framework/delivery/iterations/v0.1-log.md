## 2. 关键决策

| 决策 | 原因 |
|------|------|
| 使用 provide/inject 替代 window.* | 避免全局状态不可控，支持测试隔离 |
| 强制 noImplicitAny + no-explicit-any | 源项目 any 泄漏是首要风险 |
| fetch 替代 Alova | 降低框架层依赖，应用层可自行选择 |
| BroadcastChannel 跨 tab 同步 | 替代 localStorage event，API 更清晰 |
| 库模式 multi-entry build | 支持按模块 tree-shake |

## 3. 失败模式与修复

| 问题 | 原因 | 修复 |
|------|------|------|
| RouteRecordRaw 类型不兼容 | vue-router RouteMeta 使用 index signature | meta 类型交叉 `RouteMeta & Record<string, unknown>` |
| vitest fake timer 死循环 | `vi.runAllTimersAsync()` 递归 setInterval | 改用 `vi.advanceTimersByTimeAsync(6000)` |
| PromiseRejectionEvent 未定义 | happy-dom 不支持该全局类型 | 使用对象 mock 替代 |
