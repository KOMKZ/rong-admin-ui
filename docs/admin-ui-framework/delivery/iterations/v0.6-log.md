## 问题与修复

### 问题 1：RIcon name 类型与 MenuItem.icon 类型不兼容

**现象**：`MenuItem.icon` 类型为 `string | Component`，但 `RIcon.name` 只接受 `string`。

**修复**：在 RSideNav 中添加类型守卫 `typeof item.icon === 'string'`。

**规则沉淀**：当组件 prop 类型为 union type 时，子组件使用需添加类型守卫。

### 问题 2：RFormRenderer expose 方法名不一致

**现象**：`RDrawerForm` 调用 `formRef.value?.reset()`，但 `RFormRenderer` expose 的是 `resetFields`。

**修复**：改为调用 `formRef.value?.resetFields()`。

**规则沉淀**：调用 expose 方法前必须核对源组件的 defineExpose 定义。

### 问题 3：测试用例使用 emoji 图标不兼容

**现象**：RSideNav 测试中 `baseMenus` 使用 emoji 图标，但组件现在期望 Lucide 图标名称。

**修复**：更新测试用例，使用字符串图标名称并验证 `.r-side-nav__icon` 元素存在性。

**规则沉淀**：组件重构后必须同步更新测试用例的 fixture 数据。

## 规则沉淀

1. **图标类型守卫**：当 prop 为 `string | Component` union type 时，传递给只接受 `string` 的子组件需添加 `typeof x === 'string'` 守卫。

2. **Expose 方法命名**：调用其他组件的 expose 方法前，必须核对 `defineExpose` 定义的实际方法名。

3. **测试 Fixture 同步**：组件 API/行为重构后，测试用例的 fixture 数据必须同步更新。

4. **Design Token 命名**：
   - 背景色：`bg-surface-*`（平面）、`bg-elevated-*`（浮起）
   - 文字色：`text-primary/secondary/tertiary/quaternary`
   - 边框色：`border-default/subtle/interactive/focus`

5. **图标尺寸规范**：xs(12) / sm(14) / md(16) / lg(20) / xl(24) / 2xl(32)
