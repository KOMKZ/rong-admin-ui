# L1 hard rules (must-pass)

## Visual and interaction baseline

- Must meet modern admin-console baseline.
- No old-template signals (flat default controls, weak hierarchy, weak state feedback).
- Must support mobile and keyboard access on core paths.

## Icon system

- Single icon source only.
- Uniform icon grammar (size, stroke, corner style).
- Semantic icon coloring via tokens only.

## Component and docs baseline

- No iteration can close with only primitive component tweaks.
- Every iteration must advance commercial component set.
- Docs cannot be API-only; must include scenario, interaction, a11y, and migration guidance.

## Quality and delivery

- Gates must be command-verifiable and evidence-linked.
- Review before commit is mandatory.
- Local commit after passing gates is default mandatory behavior.
