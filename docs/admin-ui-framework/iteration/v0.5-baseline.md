# v0.5 基线分析报告

> 基于 v0.4 基线，识别可复用资产与风险模式，映射到 v0.5 迭代任务。

## 综合评分（v0.4 基线）

| 维度 | 评分 | 说明 |
|------|------|------|
| 框架工程化成熟度 | 75% | 8 模块 Contract 齐全，preset 可用，但 router pipeline / theme 缺失 |
| 组件能力成熟度 | 60% | 6 个业务组件 + 4 个展示组件，功能基础可用，缺高级能力 |
| 视觉与设计系统成熟度 | 45% | Token 散落无中心定义，无主题提供器，showcase 用 fallback |
| 真实业务可落地成熟度 | 55% | web 端 tab 切换非路由驱动，无真实后台壳层 |
| **综合** | **~60%** | 可作为 v0.5 迭代起点 |

## 模块能力地图

### 1. app-core (Bootstrap)

- **可复用**: bootstrapApp + 全局异常边界 + ErrorLogEntry
- **风险项**: destroy 路径已有清理，但无法验证所有 listener 已解绑（缺 leak 检测）
- **抽象建议**: 保持现状，补充 destroy 完整性测试
- **web 落地**: main.ts 已接入 createRongAdminApp

### 2. app-plugin

- **可复用**: pluginManager + discreteApi + 指令注册
- **风险项**: 模块级单例 discreteApi 无 reset（HMR 可能污染）
- **抽象建议**: 补 resetDiscreteApi 测试覆盖，确认 HMR 安全
- **web 落地**: 已通过 preset 隐式接入

### 3. app-layout

- **可复用**: reactive layout config（navMode / sidebar / tabs / breadcrumb）
- **风险项**: 仅暴露配置与 action，无 UI 组件（Header/Sidebar/Content）
- **抽象建议**: 需补 RAppShell / RSideNav / RTopbar / RBreadcrumb / RTabsView
- **web 落地**: config 已传入但未消费（App.vue 自建 tab 壳层）

### 4. app-router

- **可复用**: loadRouteModules + createRouteGenerator（menu→routes）
- **风险项**: 无 guard/pipeline/permission-gate 落地；RouteContract 字段部分悬空
- **抽象建议**: 补 createAdminRouterPipeline（登录重定向 / 动态注入 / 权限守卫）
- **web 落地**: 未使用 vue-router（依赖已装但无 router-view）

### 5. app-permission

- **可复用**: createPermissionService + v-permission directive（hidden/disabled）
- **风险项**: 仅在 FrameworkDemo 页演示，未与路由守卫串联
- **抽象建议**: 接入 router pipeline 做路由级鉴权
- **web 落地**: FrameworkDemo 有按钮级演示

### 6. app-request

- **可复用**: createHttpClient（fetch-based, interceptor, timeout + signal 组合取消）
- **风险项**: v0.2 已修复错误语义区分（RequestErrorKind），需保持回归
- **抽象建议**: 保持现状，回归测试已覆盖
- **web 落地**: FrameworkDemo 有 HTTP 调用演示

### 7. app-auth

- **可复用**: createTokenManager（storage / refresh / cross-tab sync / BroadcastChannel）
- **风险项**: 自动刷新幂等性已修复，需保持回归
- **抽象建议**: 保持现状
- **web 落地**: FrameworkDemo 有 login/logout 演示

### 8. app-config

- **可复用**: createAppConfig + installAppConfig + useAppConfig（project/design/runtime/website）
- **风险项**: DesignConfig 有 darkMode/primaryColor 但未接入 CSS 变量
- **抽象建议**: DesignConfig 需与 ThemeProvider 打通
- **web 落地**: 已通过 preset 接入

### 9. app-preset

- **可复用**: createRongAdminApp 一体化装配
- **风险项**: 未包含 router pipeline 和 theme provider
- **抽象建议**: v0.5 扩展为包含 theme + router
- **web 落地**: main.ts 使用中

### 10. 组件层

| 组件 | 可复用 | 风险项 | Token 状态 |
|------|--------|--------|------------|
| RDataTable | 分页/排序/选择/loading/empty | 无虚拟滚动、无列持久化 | 无 --r-* token，依赖 naive-ui |
| RFormRenderer | Schema 驱动、验证、多字段类型 | 无字段联动、upload 类型未实现 | 无 --r-* token，依赖 naive-ui |
| RModalDialog | 确认/取消/loading/自定义 footer | 功能完整 | 无 --r-* token |
| RFileUpload | 按钮/拖拽/customRequest | 无分片/断点/并发队列 | 无 --r-* token |
| RQueryTable | 查询表单 + 数据表格联动 | 可透传 DataTable 能力 | 使用 --r-* fallback |
| RCrudFormDialog | create/edit/view 三模式 | 依赖 FormRenderer 能力 | 无 --r-* token |
| Showcase 组件 | RShowcaseLayout/RDemoSection/RPropsTable/RQualityPanel | 使用 --r-* fallback | 有 token 但无全局定义 |

### 11. Token / 主题体系

- **现状**: 散落 `--r-*` CSS variable 使用 fallback 值（无 :root 定义）
- **缺口**:
  - 无 `src/theme/` 目录
  - 无 primitives.css / semantic.css
  - 无 ThemeProvider
  - 无主题预设（enterprise-blue / teal-ops / graphite-pro）
  - DesignConfig.primaryColor 未接入 CSS 变量
  - 核心业务组件（DataTable/FormRenderer/ModalDialog/FileUpload）完全依赖 naive-ui 主题
- **硬编码颜色风险**: showcase 组件的 fallback 值实质是硬编码

### 12. Story 展示体系

- **现状**: 7 个 Story 页面，仅 QueryTable 和 CrudFormDialog 使用 RShowcaseLayout
- **缺口**:
  - DataTable/FormRenderer/ModalDialog/FileUpload Story 未使用标准展示壳
  - StoryApp 标注 v0.2.0，与 package.json 0.1.0 不一致
  - E2E 测试假设 showcase-layout 但部分 story 无此结构
  - 缺少 theme-variant demo
  - 缺少 mobile breakpoint 验证

### 13. 版本治理

- **不一致**: package.json 0.1.0 / StoryApp v0.2.0 / 文档 v0.4
- **缺口**: 无 changesets / api-extractor / typedoc

### 14. 生态边界

- **正面**: gate-consumption 脚本限制 naive-ui 直引
- **风险**: `file:../rong-admin-ui` 本地引用（开发期可接受，发布需替换）
- **例外放行**: stories 和 FrameworkDemo 可直引 naive-ui（合理）
